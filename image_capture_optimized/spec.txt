Image Capture — Optimized (image_capture_optimized)
===================================================

Purpose
-------
Speed-optimized variant of the Image Capture tool. Same features and UI layout as
`image_capture/`, but tuned for higher preview frame rates on Android and faster PNG
capture. Defaults to 1280×720 camera resolution instead of maximum resolution.

Use this build when live preview responsiveness matters more than starting at the
highest sensor resolution. Users can still select Maximum or 1920×1080 in settings.


Relationship to image_capture/
------------------------------
Functional parity with the original tool:
  - Wide 16:9 preview + square center crop
  - Labeled PNG save with sequence counter and optional focus-score tag
  - Settings for camera, resolution, crop size, magnification, processing modes,
    quality metrics, barcode detection
  - Separate storage keys (does not overwrite original app settings)

Intentional differences documented in "Performance optimizations" below.


Hard constraints (must follow when rebuilding)
----------------------------------------------
1. Same as image_capture/: no ES module script tags; plain `<script src>` + inline init.
2. ZXing remains a dynamic import from esm.sh (network required for barcode ZXing modes).
3. Use separate persistence keys:
     localStorage:  image-capture-optimized-settings
     sessionStorage: image-capture-optimized-camera-options
4. Default imageResolution must be '1280x720', not 'max'.
5. Save must encode from #saveCanvas (clean output), never from #zoomCanvas (overlays).
6. Preview pixel processing runs at reduced resolution; save always uses full crop size.
7. Focus score WASM must be embedded in JS (SINGLE_FILE=1) — separate .wasm fetch fails
   on file://. Run ./build-wasm.sh after editing focus-score.cpp.

Performance optimizations (vs image_capture/)
-----------------------------------------------

Default camera resolution
  DEFAULT_IMAGE_RESOLUTION = '1280x720'
  Reduces per-frame pixel count on Android for smoother preview. User can switch to
  Maximum or 1920×1080 in settings (restarts camera on return).

Center preview frame cap
  MAX_CENTER_FPS = 30
  Center crop updates at most 30 times per second. Uses video.requestVideoFrameCallback
  when available; otherwise setTimeout fallback.

Throttled quality metrics
  METRICS_INTERVAL_MS = 250
  Focus score and histogram computed at most ~4 Hz. Displayed values use the rolling
  1 s focus average between updates.

Focus score WebAssembly
  computeFocusScore() prefers WASM when FocusScoreModule loaded; falls back to
  computeFocusScoreJs() on load failure or missing emscripten build.
  initFocusWasm() runs at camera init (parallel with stream open).
  Algorithm matches JS: RGBA center sample → grayscale (BT.601) → 3×3 Gaussian blur
  (edge clamp) → 4-neighbor Laplacian histogram → 20th-ranked value by count.
  Input region is at most 200×200 (FOCUS_REGION_SIZE); WASM heap 16 MB fixed.

Half-resolution preview processing
  PREVIEW_PROCESS_SCALE = 0.5, PREVIEW_PROCESS_MIN_SIZE = 256
  edgeMap, movingAverage, changeDetection:
    - Preview: draw crop → downscale to process canvas → getImageData at reduced size
      → process → upscale to zoom canvas (imageSmoothingEnabled = false)
    - Save: full cropSize × cropSize processing on #saveCanvas (no downscale)
  Zoom label appends " · preview ½×" when a pixel-processing mode is active.

Dedicated save path (major capture speedup)
  Original re-ran full renderCenterView (metrics, overlays, preview processing) then
  encoded zoomCanvas.

  Optimized savePng():
    1. Single video → cropCanvas draw at full crop size
    2. renderSaveFrame() → #saveCanvas (full-res processed or raw RGB)
    3. Focus filename tag from cached getSmoothedFocusScore() — no recomputation
    4. canvasToPngBlob(saveCanvas) — prefers OffscreenCanvas.convertToBlob
    5. Fallback download uses URL.createObjectURL(blob), not toDataURL (avoids main-thread
       base64 encoding of large PNGs)

  Barcode overlay is preview-only; saved PNG is raw crop (no detection graphics).

Canvas context tuning
  wideCtx / zoomCtx: { alpha: false, desynchronized: true }
  cropCtx / saveCtx / previewProcessCtx: willReadFrequently: true

Wide view overlay cadence
  WIDE_OVERLAY_INTERVAL_MS = 2
  Dashed center-crop rectangle redrawn every 2 wide frames (video still every frame).

Barcode detection
  BARCODE_DETECT_INTERVAL_MS = 800 (was 600)
  BARCODE_DETECT_MAX_EDGE = 640 — downscale source before detect; scale box back

Cached center crop
  Wide loop stores cachedCenterCrop; center loop reuses it to avoid duplicate math.

  PREVIEW_PROCESS_MIN_SIZE = 256

Render loops (optimized)
------------------------
Wide loop: unchanged rAF cadence; overlay every 2 frames; updates cachedCenterCrop.

Center loop:
  scheduleNextCenterFrame() → queueCenterFrame(now)
  Skip if centerInFlight or elapsed < CENTER_FRAME_MIN_MS
  renderCenterView with throttled metrics and half-res preview processing

startRenderLoops resets wideFrameCounter, lastCenterFrameMs, lastMetricsUpdateMs.


Save flow
---------
savePng():
  video → cropCanvas (full crop)
  renderSaveFrame():
    none / barcode: video → saveCanvas
    edgeMap / movingAverage / changeDetection:
      cropCanvas → renderProcessedSave() at full squareSize
  focus tag from getSmoothedFocusScore() if enabled
  blob = await canvasToPngBlob(saveCanvas)
  showSaveFilePicker or object-URL download

Focus score WASM (focus-score.cpp / build-wasm.sh)
--------------------------------------------------
Exported C functions:
  init_focus_buffers(width, height)   — pre-allocate RGBA + float workspaces
  get_focus_input_buffer()            — uint8_t* RGBA in
  compute_focus_score(w, h, top_rank) — returns double score

Build (requires em++):
  ./build-wasm.sh

  em++ focus-score.cpp -O3 -std=c++17 \
    -s WASM=1 -s SINGLE_FILE=1 \
    -s MODULARIZE=1 -s EXPORT_NAME='FocusScoreModule' \
    -s EXPORTED_FUNCTIONS='["_init_focus_buffers","_get_focus_input_buffer","_compute_focus_score","_malloc","_free"]' \
    -s EXPORTED_RUNTIME_METHODS='["HEAPU8"]' \
    -s ALLOW_MEMORY_GROWTH=0 -s INITIAL_MEMORY=16777216 \
    -o focus-score-wasm.js

Prepends: globalThis.FOCUS_SCORE_WASM_BUILD_ID="<YYYYMMDDHHMMSS>";

Rebuild checklist
-----------------
1. Copy image_capture/ structure into image_capture_optimized/.
2. Apply settings.js changes (new keys, 1280×720 default, perf constants).
3. Add preview/save processing helpers in image-processing.js.
4. Implement focus-score.cpp; run ./build-wasm.sh → focus-score-wasm.js.
5. Add focus-wasm-bridge.js; wire computeFocusScore WASM/JS in image-quality.js.
6. Add shouldUpdateMetrics + canvasToPngBlob in image-quality.js.
7. Add downscaled barcode detect in barcode-detection.js.
8. Wire optimized loops and save path in image_capture.html:
   - Script order includes focus-score-wasm.js + focus-wasm-bridge.js
   - initFocusWasm() in init()
   - #saveCanvas, #previewProcessCanvas
   - 30 FPS center cap, 250 ms metrics throttle
   - Half-res preview, full-res save
9. Update settings.html title; keep separate storage key.
10. Add index.html link; write this spec.txt.
11. Verify: focus score with WASM loaded; JS fallback if wasm file missing;
    Android preview at 720p; fast save; full-res PNG output.


Operational notes
-----------------
- First launch uses 1280×720 — change to Maximum in settings for full sensor res.
- Preview processing modes look softer/blockier than save output (half-res preview).
- Focus score in filename reflects the rolling average, not a one-off recomputation.
- Original image_capture/ settings are independent (different localStorage key).
- After editing focus-score.cpp: run ./build-wasm.sh, hard-refresh capture page.
- Check FOCUS_SCORE_WASM_BUILD_ID in focus-score-wasm.js if WASM scores look stale.
