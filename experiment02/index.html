WASM vs JavaScript

What is this experiment?

This app uses your device camera and turns the live video into a black-and-white edge drawing — like tracing the outlines of people, objects, and shapes in the scene. The picture you see is the same in all three tests; only the way the computer calculates those edges changes.

We run that same job three different ways to compare speed:

  • JavaScript — the usual language web pages are written in.
  • WebAssembly — a faster, compiled version of the same steps.
  • Full Optimization — WebAssembly tuned for maximum speed.

Higher FPS (frames per second) means smoother, faster processing. After you try each test, open Comparison summary to see the numbers side by side.

How to run each experiment

  1. Tap one of the three tests below (start with any — the steps are the same).
  2. Allow camera access when your browser asks.
  3. You will see live video from your camera. Note the Display FPS while processing is off.
  4. Tap Turn Processing ON in the top-left corner. The view switches to the edge drawing.
  5. Watch Processing FPS — that is how fast the edge effect runs.
  6. Use the ← back arrow to return here, then repeat for the other two tests.
  7. Open Comparison summary to compare your latest results from all three tests.
